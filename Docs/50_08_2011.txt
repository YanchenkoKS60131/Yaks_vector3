Единая информационная система ОАО «Сургутнефтегаз»

СТО 50.8 – 2011
Введен впервые
	Дата введения __________________
    1. Область применения
        1.1 Настоящий стандарт устанавливает порядок работ по проведению системного мониторинга в единой информационной системе (далее по тексту – ЕИС) открытого акционерного общества «Сургутнефтегаз» (далее по тексту – Общество).
1.2 Стандарт обязателен для всех структурных подразделений Общества, а также управлений, отделов и служб аппарата управления Общества.
    2 Нормативные ссылки
В настоящем стандарте использованы ссылки на следующие стандарты:
СТО 50.0-2011 Единая информационная система ОАО «Сургутнефтегаз». Общие положения
СТО 50.1-2011 Единая информационная система ОАО «Сургутнефтегаз». Документирование ИТ-решений
СТО 50.4-2011 Единая информационная система ОАО «Сургутнефтегаз». Управление инцидентами
    3 Термины и определения
В настоящем стандарте применены термины с соответствующими определениями:
        3.1 объект мониторинга: Компонент инфраструктуры, начиная с целой системы, включающей аппаратные средства, программные средства и документацию, заканчивая отдельным модулем или небольшим аппаратным компонентом. 
        3.2 системный мониторинг: Процесс периодического и (или) непрерывного сбора информации о показателях объекта мониторинга для определения тенденций изменения параметров.
        3.3 другие термины, определения которых даны в СТО 50.0.
    4 Основные положения
        4.1 Целью системного мониторинга является повышение эффективности и надежности эксплуатации ИТ-решений для удовлетворения потребностей бизнеса.
        4.2 Основными задачами системного мониторинга являются:
    • определение тенденций изменения параметров мониторинга при эксплуатации ИС, в том числе для своевременного выявления фактов нарушения  функционирования ИС;
    • сбор и хранение данных системного мониторинга;
    • оперативное информирование соответствующих служб для восстановления работоспособности ИС.
        4.3 В процессе системного мониторинга задействованы следующие элементы:
    • система мониторинга;
    • объекты мониторинга;
    • индикаторы (показатели мониторинга и пороговые значения), позволяющие определять состояние объекта мониторинга.
        4.4 Система мониторинга должна иметь предустановленные, либо устанавливаемые основные индикаторы (правила), позволяющие определять и измерять состояние объектов мониторинга.
        4.5 Объектами мониторинга могут быть:
    • ИТ-решения;
    • ИС;
    • компоненты ИТ-инфраструктуры;
    • технические компоненты инфраструктуры (сервера, персональные компьютеры, система хранения данных, сети, сетевое оборудование, устройства ввода/вывода информации и др.);
    • ПО (приложения, операционные системы, инструментальные среды разработки, средства обеспечения информационной безопасности и др.).
        4.6 Постановка на системный мониторинг производится на основании утвержденной Программы выполнения системного мониторинга (приложение А), в которой должны быть определены все основные элементы процесса системного мониторинга, а также его особенности.
    5 Участники процесса системного мониторинга
        5.1 В процессе анализа и контроля разрабатываемого ПО участвуют группы ролей:
– «Управление ИТ-решениями»;
– «Управление ИT-инфраструктурой»;
– «Системное администрирование».
        5.2 Процесс анализа и контроля разрабатываемого ПО осуществляют специалисты:
– для групп ролей «Управление приложениями», «Системное администрирование», «Управление ИТ-инфраструктурой» «Управление ИТ-решениями» - Заказчик, определяющий  объекты и показатели мониторинга;
– для группы ролей «Системное администрирование» - Администратор системного мониторинга, принимающий решение о выборе инструментов и методов мониторинга, выполняющий настройку системы мониторинга;
– для группы ролей «Управление ИT-инфраструктурой» - Оператор мониторинга, выполняющий системный мониторинг.
Примечание - Приведены типовые роли из групп ролевой модели, закрепленной СТО 50.0.
    6 Процесс  системного мониторинга
	Схематично процесс системного мониторинга и его взаимодействие с другими стандартами отражен на рисунке 1.

Рисунок 1 – Схема процесса системного мониторинга 
	6.1 Определение объектов мониторинга, показателей и пороговые значения
            6.1.1 Заказчик совместно с Администратором системного мониторинга определяют объекты и показатели мониторинга. 
            6.1.2 Для объектов мониторинга, на основании ПТД (СТО 50.1), паспортной документации и индивидуальных требований к компонентам 
ИТ-инфраструктуры, определяются показатели и их пороговые значения. 
            6.1.3 Показатели могут относиться к следующим основным группам:
	- доступность (показатель, определяющий способность компонентов ИТ-инфраструктуры выполнять свои функции в определенный период времени);
	- пропускная способность (показатель, характеризующий в числовом выражении соотношение количества проходящих единиц информации в единицу времени через канал, систему, узел);
	- особые ситуации (показатель, характеризующий в числовом выражении количество системных ошибок, критичных событий системы);
	- безопасность (показатель, определяющий наличие возможности неавторизованного доступа, IP-маска допущенных рабочих мест, прав и ролей пользователей);
	- производительность (показатель, характеризующий степень загрузки системных ресурсов при нормальной работе, время отклика на действия пользователей и смежных процессов);
	- надёжность (показатель, характеризующий параметры отказо-устойчивости, периодичность резервного, разностного и полного копирования, доступность ресурсов для оперативного восстановления работоспособности).
6.2 Подготовка программы выполнения системного мониторинга 
Администратор системного мониторинга совместно с Заказчиком выполняет анализ объектов мониторинга, определяет их взаимозависимости, принимает решение о выборе инструментов и методов мониторинга, совместно с Заказчиком подготавливает Программу выполнения системного мониторинга (приложение А).  
6.3 Настройка системного мониторинга
6.3.1 Администратор системы мониторинга выполняет настройку системы  в соответствии с Программой и решением о выборе инструментов и методов мониторинга по объекту мониторинга и подготавливает инструкцию по настройке системного мониторинга (приложение Б).
6.3.2 Системный мониторинг может выполняться в следующих режимах: 
- автоматический, выполняется без непосредственного участия оператора мониторинга, согласно настроенному ранее сценарию, в соответствии с установленными регламентами. Данный метод основан на методах реакции на достижение пороговых значений показателей мониторинга и является наиболее предпочтительным;
- по требованию, выполняется оператором мониторинга при возникновении потребности с применением инструментов для анализа состояния объектов мониторинга;
- периодический, выполняется оператором мониторинга, в соответствии с установленным регламентом.

	6.4 Мониторинг и сбор статистики
	6.4.1 Системный мониторинг выполняется оператором мониторинга в режиме, определённом в Программе выполнения системного мониторинга (приложение А).
            6.4.2 В процессе системного мониторинга и анализа данных информация мониторинга передается всем заинтересованным службам в соответствии с СТО 50.4, согласно действующим инструкциям и регламентам.
            6.4.3  Администратор системы мониторинга производит анализ данных мониторинга и, при необходимости, вносит изменения в Программу выполнения системного мониторинга (приложение А) и инструкцию по настройке (приложение Б).
