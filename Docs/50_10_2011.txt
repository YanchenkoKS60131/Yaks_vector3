1 Область применения 
1.1 Настоящий стандарт устанавливает порядок организации работ по управлению объёмами данных (жизненным циклом данных) в единой информационной системе (ЕИС) открытого акционерного общества ОАО «Сургутнефтегаз» (далее по тексту – Общество).
	1.2 Требования настоящего стандарта распространяются на все внедряемые и эксплуатируемые в ЕИС ИТ-решения независимо от их сложности и способа реализации.
	1.3 Стандарт обязателен для всех структурных подразделений Общества, а также управлений, отделов и служб аппарата управления Общества.

3 Термины и определения
В настоящем стандарте применены следующие термины с соответствующими определениями:
        3.1 база данных; (БД): Совместно используемый набор логически связанных данных (и описание этих данных), предназначенный для удовлетворения информационных потребностей организации.
        3.2 исторические данные: Данные, которые утратили свою необходимость использования в продуктивной БД ИС, но при необходимости могут в дальнейшем использоваться.
        3.3 архивирование: Процесс сжатия и перемещение  исторических данных в архивный файл для их длительного хранения.
        3.4 объект архивирования: Совокупность данных с сохранением их целостности и возможности последующей работы с ними.
        3.5 жизненный цикл данных: Период времени, который начинается с момента принятия решения о необходимости создания данных и заканчивается в момент их уничтожения. 
        3.6 управление жизненным циклом данных: Комплекс организационных и аппаратно-программных решений, обеспечивающих эффективное управление процессами манипулирования данными на протяжении жизненного цикла данных.
        3.7 система хранения данных; (СХД): Комплексное программно-аппаратное решение по организации надёжного хранения архивных данных и предоставления гарантированного доступа к ним.
        3.8 время хранения: Определяемое на основе правовых требований время, указывающее минимальный срок хранения данных в продуктивной  ИС и СХД. 
        3.9  топ(x): Список показателей рейтинга объекта БД, отсортированный по значимости  в управлении объёмами данных, где x – число объектов. 
        3.10  фильтрация данных: Отсеивание данных, в которых нет необходимости для повышения достоверности и адекватности. 
Примечание - Для фильтрации поступающих в БД данных применяются коэффициенты вариации. 
        3.11  коэффициент вариации: Величина изменения последнего значения какого-либо признака данных от предыдущего.
        3.12  другие термины, определения которых даны в СТО 50.0.
4 Общие положения
	Основными задачами процесса управления объёмами данных являются:
- систематизация подхода к организации централизованного процесса управления объёмами данных в ЕИС Общества; 
- повышение эффективности использования компонентов ИТ-инфра-структуры (дисковых устройств, процессоров, памяти) и снижение затрат на эксплуатацию и сопровождение;
- оптимизация дискового пространства за счёт фильтрации поступающих в БД оперативных данных, архивирования (или удаления) из БД исторических данных, перемещения архивных данных в СХД;
- сокращение времени на резервное копирование и восстановление ИС.
5 Участники процесса управления объёмами данных	
	5.1 В процессе управления объёмами данных участвуют следующие группы ролей:
	- «Управление ИТ-решениями»;
	- «Управление приложениями»;
	- «Управление разработками»;
	- «Управление ИТ-инфраструктурой»;
	- «Системное администрирование».
	5.2 Ответственными за процесс управления объёмами данных являются специалисты:
а) для группы ролей «Управление ИТ-решениями»:
1) Менеджер процесса, обеспечивающий организацию процесса управления объёмами данных, согласовывающий применение методик, требований к данным, а также контролирующий обеспечение (наличие) регламентов и ПТД по управлению объёмами данных в ИТ-решениях ЕИС;
б) для группы ролей «Управление приложениями»:
1) Эксперт по управлению объемами данных, определяющий методики, требования к данным в части организации оптимального процесса управления объёмами и показателями;
в) для группы ролей «Управление разработками»:
1) Разработчик ПО, выполняющий разработку или доработку программного обеспечения по управлению объёмами данных в соответствии с ПТД;
г) для группы ролей «Управление ИТ-инфраструктурой»:
1) Оператор, отвечающий за выполнение операций по управлению объёмами данных;
д) для группы ролей «Системное администрирование»: 
1) Системный администратор, отвечающий за организацию доступа к информационным ресурсам (БД ИС, СХД), настройку мониторинга показателей, выполнение анализа и реорганизации БД ИС, анализ операций по управлению объёмами данных.
Примечание - Приведены типовые роли из групп ролевой модели, закрепленной СТО 50.0.
6 Требования к управлению объёмами данных
6.1 На этапе проектирования ИТ-решений Экспертом по управлению объемами данных прорабатывается стратегия управления с учетом жизненного цикла данных и методик по управлению объемами данных.
6.2 Соответствующие требования фиксируются в ПТД (СТО 50.1) или технических требованиях к решению, в случае заказа решения у стороннего производителя, с учетом требований к программному и аппаратному обеспечению.
6.3 В случае приобретения готовых решений у сторонних производителей, применяется стратегия управления объёмами данных, разработанная производителем.
6.4 Контроль объема БД в процессе эксплуатации решений осуществляется в рамках системного мониторинга (СТО 50.8).
6.5 В случае превышении предусмотренного объёма БД, для обеспечения эффективности работы ИТ-решений прорабатывается вопрос дополнительной технической возможности увеличения объёма и производительности дискового пространства.
7 Этапы процесса управления объёмами данных 
	Процесс управления объёмами данных и его взаимодействие с другими процессами по операционной деятельности схематично изображены на рисунке 1. 

Рисунок 1 – Процесс управления  объёмами данных и его взаимодействие с другими процессами
        7.1 Анализ БД
	На основании информации, полученной в результате системного мониторинга (СТО 50.8) выполняется: 
	- определение топ(x) объектов БД, занимающих наибольший объём дискового пространства или показывающих наибольший рост объёма данных; 	- определение предварительных требований к данным и их взаимозависимостей;
	- анализ отдельных таблиц или объектов БД на предмет роста данных;
	- классификация данных для определения конкретных областей БД, которые подвержены быстрому росту; 
	- определение причин роста объёма данных;
	- прогнозирование темпов роста объёма данных;
	- предварительный расчёт объёма данных, высвобождаемого в процессе архивирования/удаления данных и объёма архивных данных, предназначенного для хранения в СХД. 
        7.2 Определение методики управления
После анализа БД, при необходимости, осуществляется:
- определение необходимости архивирования, удаления или фильтрации данных;
- определение объектов архивирования;
- определение (расчёт) коэффициентов вариации;
- выбор, разработка, настройка и тестирование методик управления объёмами данных.  
        7.3 Согласование жизненного цикла данных
7.3.1 В проектных решениях (СТО 50.1) определяются параметры управления объёмами данных:
- перечень используемых в ИТ-решении таблиц или объектов БД;
- критерии выборки или фильтрации данных из БД (в соответствии с методикой управления данными);
- время хранения данных в БД и СХД;
- расчётная величина ежегодного прироста объёма данных в БД.
7.3.2 В соответствии с требованиями к данным, определённым на этапе анализа БД (см. 7.1), методикой (см. 7.2) и параметрами управления объёмами данных выполняется:
- определение и согласование политик жизненного цикла данных (время хранения, использования в отчётности, уничтожения данных);
- регламентация процесса управления объёмом данных (приложение А).
        7.4 Выполнение операций
На этапе выполнения операций в соответствии с выбранной методикой управления объёмами данных выполняется: 
- согласование временных интервалов (с учётом временных и аппарат-ных ограничений и производительности ИС) для планирования запуска заданий в соответствии с требованиями, определёнными СТО 50.9;
- расчёт производных (аналитических) параметров и настройка коэффициентов вариации для фильтрации данных; 
- предварительная подготовка данных к архивированию (установка меток удаления или архивирования);
- запуск заданий по архивированию и удалению данных с последующей реорганизацией БД, проверка выполнения операций на наличие ошибок (СТО 50.4);
- перенос архивных файлов в СХД.
        7.5 Анализ результатов
По результатам выполненных работ по управлению объёмами данных производится:
- сбор статистики по выполненным операциям;
- формирование акта выполненных работ по установленной в Обществе форме.
